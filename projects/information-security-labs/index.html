Information Security - 安全实验整理

项目概览

Information Security 课程材料包含多次实验报告、模板和实验记录。这个板块主要用于整理本科阶段信息安全课程中的工具使用、实验过程和安全分析方法。

学习重点

信息安全课程通常覆盖密码学、系统安全、网络安全和安全工程基础。实验关注的不只是“攻击是否成功”，还包括漏洞形成原因、攻击路径、风险影响和防护思路。

实验记录方式

安全实验报告通常需要包含：

  • 实验目标。
  • 环境配置。
  • 工具和命令。
  • 攻击或验证步骤。
  • 关键截图或日志。
  • 原因分析。
  • 防护建议。

这种结构让我意识到安全工作必须可复现、可解释，而不是只展示最终结果。

方法沉淀

课程实验训练了几个习惯：

  • 对环境和版本保持敏感。
  • 记录每一步输入输出。
  • 区分现象、原因和影响。
  • 在结论中同时考虑攻击者视角和防御者视角。

这些习惯在后续调试复杂工程项目时也很有用。

后续整理方向

由于安全实验可能包含具体漏洞、环境细节或截图，公开页面暂时只保留课程项目摘要。后续如果要公开详细实验，需要先清理敏感信息，避免暴露账号、内部环境或可被滥用的细节。
