SCNN CUDA Inference Optimization

Project Overview

This project optimized CUDA inference for spiking convolutional neural networks. The original workload had a long inference time and a timestep-heavy computation pattern, so the main challenge was to redesign execution around GPU parallelism, memory reuse, and reduced launch overhead.

The optimized version reduced inference time from 5.64s to 15.4ms while maintaining 89.89% accuracy.

Optimization Methods

The system used batching and pipelining to improve parallelism across samples and computation stages. Ping-Pong buffering helped overlap data movement and computation, reducing idle time between stages.

For compute kernels, I explored WMMA Tensor Core programming and PTX inline instructions. CUDA Graph was used to reduce repeated kernel-launch overhead, which is especially useful when the inference flow contains many small or repeated operations.

Model and Accuracy Considerations

Because spiking neural networks depend on timestep behavior, optimization could not only focus on raw kernel speed. I also considered timestep-aware retraining and accuracy preservation so that acceleration did not destroy the model’s inference quality.

This made the project a system-level optimization task: kernel design, execution scheduling, graph capture, memory buffering, and model behavior all affected the final result.

Takeaways

SCNN CUDA optimization gave me practical experience with GPU performance engineering. The most valuable lesson was that high speedups usually come from combining several layers of optimization: algorithm-level simplification, memory hierarchy awareness, Tensor Core usage, launch-overhead reduction, and model-aware accuracy control.
