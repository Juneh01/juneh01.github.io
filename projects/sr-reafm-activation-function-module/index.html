SR - ReAFM 可重构激活函数模块调研

项目概览

ReAFM 方向关注神经网络中非线性激活函数的硬件实现。项目材料中包含 ReAFM 论文、不同版本实现目录、MinusExp、ReLU、OPart 等模块，以及高精度、可重构、对称性感知的 element-wise activation function 硬件架构相关论文。

它属于硬件友好神经网络方向：不是研究新的深度模型结构，而是研究神经网络中的常用算子如何在硬件上更高效、更灵活地实现。

背景问题

神经网络硬件加速器中，卷积、矩阵乘法等线性计算通常最受关注，但激活函数同样会影响整体效率。尤其是 sigmoid、tanh、GELU、SiLU 这类非线性函数，如果直接用高精度数学函数实现，会带来面积、功耗或延迟成本。

常见实现策略包括：

  • 查找表。
  • 分段线性近似。
  • 多项式近似。
  • CORDIC 或指数函数近似。
  • 针对特定激活函数的专用电路。

ReAFM 方向关注的是是否能用一个可重构模块支持多类激活函数，从而减少为每个函数单独设计硬件的成本。

技术关注点

可重构性

可重构激活函数模块希望在同一套硬件中支持 ReLU、sigmoid、tanh、GELU 或指数相关函数。这样在不同网络层或不同模型部署时，可以通过配置切换函数形式。

这类设计需要解决的问题是：通用性提高后，面积和延迟是否仍可接受。

精度与硬件成本

激活函数近似必须在精度和成本之间取舍。误差过大会影响网络推理质量，精度过高又会增加硬件复杂度。

因此，调研中关注：

  • 近似误差如何度量。
  • 不同位宽下误差和硬件成本的关系。
  • 对称性是否可以减少计算量。
  • 函数分段方式如何影响 LUT 大小和插值复杂度。

Element-wise 算子硬件化

激活函数本质上是 element-wise operation。它不需要复杂数据复用，但会大量出现在网络中。因此单个算子延迟看似不大，累积后仍可能影响 pipeline。

这个方向让我把神经网络中的“函数”看成硬件模块，而不只是 Python 里的 API。

我的工作与沉淀

在这个方向上，我主要做了论文调研、代码/版本整理和不同 ReAFM 实现形式的理解。目录中保留了 ReAFM_v1、ReAFM_v2、ReAFM_ReLU、ReAFM_MinusExp、ReAFM_OPart 等阶段性材料。

虽然这部分不像 FYP 那样形成完整系统实验，但它帮我建立了神经网络硬件加速的算子级视角：模型压缩、量化和硬件算子设计本质上都在回答“如何用更少资源表示足够好的函数”。

后续方向

如果继续深入，可以补充：

  • 不同激活函数近似方法的误差曲线。
  • Verilog/VHDL 或 HLS 实现对比。
  • 面积、功耗、时延和精度的统一评估表。
  • 与量化神经网络部署的结合。
