Final Year Project - Lightweight 6DoF Pose Estimation

项目概览

本科毕业设计题目是 Design of a Lightweight 6DoF Pose Estimation Algorithm。项目围绕 RGB-D 物体六自由度姿态估计展开，目标是在 HiPose 框架基础上探索更轻量、更高效的网络结构，使模型在保持姿态估计精度的同时减少不必要的计算。

这个项目的核心方案被整理为 HiPose-DG：在原始 HiPose 架构中加入动态空间门控和动态通道门控，让网络根据输入特征自适应选择更重要的空间区域和特征通道。

问题背景

6DoF 姿态估计要预测物体坐标系到相机坐标系之间的刚体变换，也就是旋转和平移。它是机器人抓取、增强现实、自动驾驶和工业检测中的基础问题。

这个任务的困难主要来自：

  • 单目 RGB 图像存在深度歧义。
  • 目标物体可能低纹理、遮挡严重或形状相似。
  • RGB-D 方法虽然引入了几何信息，但网络计算和后处理可能变得昂贵。
  • 传统姿态估计常依赖 RANSAC、ICP 或渲染式 refinement，速度和稳定性都可能受影响。

HiPose 的优势在于使用层次化二进制表面编码，把 RGB-D 输入中的点与物体表面从粗到细建立对应，并通过逐步去除离群点替代昂贵的 RANSAC 过程。

技术路线

HiPose 基础框架

HiPose 的核心思想可以概括为：

  • 对 3D 物体模型表面进行层次化二进制编码。
  • 使用 RGB-D 网络预测输入点云和物体模型表面之间的 3D-3D 对应。
  • 先用粗粒度编码估计初始姿态。
  • 再逐步使用更细粒度编码，通过点到表面距离剔除离群点。
  • 最终得到不依赖额外 refinement 的 6DoF 姿态。

这个框架本身已经比很多依赖迭代渲染或 RANSAC 的方法更直接，但网络前向和特征融合仍有进一步轻量化空间。

Dynamic Spatial Gating

动态空间门控的目标是让模型把计算重点放在更有判别力的空间区域。例如，在目标物体边缘、局部几何变化明显、遮挡交界处，特征往往更关键；而背景或冗余区域可以减少计算权重。

项目中将空间门控接入 PSPNet decoder，让 decoder 在恢复高分辨率特征时能够对不同空间位置分配不同重要性。

Dynamic Channel Gating

动态通道门控关注的是特征通道维度。不同输入图像或点云下，并不是每个通道都同等重要。通道门控希望选择更具判别性的特征通道，降低冗余计算。

项目中将通道门控与 ConvNeXt backbone 结合，探索在 backbone 表达能力和计算效率之间的平衡。

Loss 与训练配置

为了避免门控机制简单地关闭过多特征导致精度下降，项目中设计了与门控相关的损失项。训练目标不只是姿态估计准确，也要鼓励模型形成合理的计算稀疏性。

技术手册中记录了 Docker 环境、BOP 数据集准备、LMO 测试流程、训练命令、测试命令、checkpoint 记录和不同物体类别上的结果。

实验内容

主要实验集中在 LMO 数据集的多个物体类别上，例如 ape、can、cat、driller、duck、eggbox、glue 和 holepuncher。记录中对比了 original 与 revised PSPNet/ConvNeXt 版本的 checkpoint 和评估表现。

实验过程也暴露出一个重要问题：轻量化并不是单调收益。某些类别可能因为形状、遮挡或纹理特征不同，对门控更加敏感。因此轻量化策略需要和任务数据特性结合分析。

我的收获

这个项目把论文理解、代码修改、训练环境、数据集结构、GPU 实验和结果分析串在了一起。相比单纯阅读 HiPose，我真正体会到一个视觉模型从想法到可运行实验之间有大量工程细节：Docker 环境、CUDA 版本、数据目录、GT 生成、checkpoint 管理、评估脚本和结果复核缺一不可。

后续可以继续补充的方向包括：更系统的 FLOPs/latency profiling、更稳定的门控损失权重搜索、不同 backbone 的对比，以及在更多 BOP 数据集上的泛化测试。
