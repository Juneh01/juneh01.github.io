SR - 模型剪枝与压缩调研

项目概览

SR 中的 Pruning / CrAM 板块关注模型压缩和压缩感知优化。材料中包含 CrAM 相关论文、不同版本代码目录和剪枝方向 review。它与 AICAS、ReAFM、DeFiNES 属于同一条大线：如何让神经网络在有限资源下更高效运行。

背景问题

深度神经网络通常包含大量冗余参数。剪枝的目标是删除不重要的权重、通道、注意力头或结构单元，在尽量保持精度的同时减少计算量和存储成本。

常见剪枝方向包括：

  • 非结构化权重剪枝。
  • 结构化通道剪枝。
  • block pruning。
  • 动态稀疏。
  • 训练时压缩感知优化。

剪枝的难点不是“删掉参数”本身，而是如何判断哪些参数可以删、删完之后如何恢复精度，以及稀疏结构能否真正被硬件加速。

CrAM 方向

CrAM 的核心关注点是 compression-aware minimization。它试图在训练或优化过程中考虑模型被压缩后的表现，而不是训练完后再被动剪枝。

这类方法的直觉是：如果一个模型在优化阶段就被引导到更适合压缩的参数区域，那么后续剪枝或量化造成的性能损失可能更小。

调研重点

我在这个方向上主要关注几个问题：

  • 剪枝指标如何定义，例如权重大小、梯度、Hessian 近似或 activation sensitivity。
  • 剪枝后是否需要 fine-tuning。
  • 稀疏比例和精度下降之间如何权衡。
  • 结构化剪枝和硬件实际加速之间的关系。
  • 压缩感知训练如何影响模型最终落点。

与其他项目的联系

这个方向和其他本科项目之间有明显关联：

  • 和 AICAS 的低比特量化共同服务于端侧部署。
  • 和 DeFiNES 一样关注硬件友好的模型执行。
  • 和 ReAFM 一样站在神经网络系统实现角度理解模型。
  • 和 FYP 的动态门控也有相似思想：不是所有特征都值得同等计算。

我的收获

剪枝调研让我更清楚地认识到，模型效率不只是一个训练后压缩步骤，而是一种贯穿模型设计、训练、部署和硬件映射的思路。

后续如果继续做，可以加入具体模型上的剪枝实验，例如比较剪枝前后参数量、FLOPs、latency 和 accuracy 的变化，并验证稀疏模式是否能真正带来实际推理加速。
